**Testing Strategy**

**Unit Testing**

File: test.py

Purpose: To test the interaction between the client and the server.

Steps:
First we have to run server.py on one terminal.
Then, Run the following command in git bash to execute the client tests:

python test.py

Validate the results displayed in the terminal.

**End-to-End (E2E) Testing
**
**File: test_e2e.py**

Purpose: To perform an end-to-end integration test, ensuring that the client and server interact correctly with the deployed service logic.

Steps:
First we have to run server.py on one terminal.
Run the following command to execute E2E tests:

python test_e2e.py

Check the output for successful integration.
